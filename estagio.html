Relatório de estágio

Introdução

O relatório apresentado tem como objetivo discorrer sobre o estágio realizado na empresa “hlh-Assessoria e Consultoria”, no período de 03/01/23 a 20/04/23 na carga horária de 168 horas. O estágio realizado tem como finalidade garantir ao estudante a prática de seus conhecimentos obtidos no curso e cumprir com as exigências do curso técnico em informática para a obtenção do certificado de conclusão.

Objetivos

Objetivo geral

  • Cumprir com as exigências do ifnmg para obtenção do certificado de conclusão.

    • Objetivos específicos

  • Aprimorar os conhecimentos obtidos no curso técnico em informática.
  • Conseguir experiência no mercado de trabalho.
  • Obter experiência na área de técnico em informática.

    • Atividades realizadas

    Java

    Ao começar, a principal habilidade desenvolvida foi a programação em linguagem Java, cuja foi muito importante para a complementação ao aprendido em sala . Primeiramente foi trabalhado a orientação a objetos, aprendendo os conceitos básicos, como o funcionamento de métodos construtores, para a construção de um objeto e a definição de seus atributos, polimorfismo de um objeto e a herança entre classes, compreendendo os conceitos e o uso da sobrescrita e sobrecarga de métodos, superclasse, classe mãe e classe filha. Logo depois, foi trabalhado o controle de exceções, entendendo como tratar exceções geradas pelo código através dos blocos try e catch, como lançar pelo throw e como indicar uma possível exceção em um método por meio do throws.

    Java web

    Uma das principais habilidades trabalhadas durante o estágio foi Java para web. Sendo essa, a habilidade com maior dificuldade de aprendizado, devido a pouca interação e conhecimento na área de web, no entanto, ao longo de seu desenvolvimento, tais dificuldades foram contornadas. Ao decorrer do desenvolvimento dessa habilidade foi aprendido o funcionamento de tecnologias fundamentais para o desenvolvimento em Java web, como o servidor apache tomcat, que fornece um ambiente que permite a execução da aplicação. Padrão MVC, utilizado para deixar a estrutura da aplicação segura, dinâmica e organizada. Classe servlet para a criação de páginas dinâmicas na aplicação. Classe filter, utilizado para a criação de filtros na aplicação. Framework PrimeFaces, utilizado para adicionar uma variedade de componentes a interface da aplicação. Entre outras.
